# H·HFGAT — Fashion Recommendation Frontend

---------
**HELLO MY TEAM!**
---------




A **Next.js 16** web application that provides an interactive demo for the **Hierarchical Fashion Graph Attention Network (H-FGAT)** model. The frontend communicates with a Python/Flask backend (deployed on Render) via a transparent API proxy.

---

## Available Features

| # | Feature | Route | Description |
|---|---------|-------|-------------|
| 1 | **Recommend Personalized Outfit** | `/recommend` | Browse users, view their interaction history, and get top-10 outfit recommendations generated by H-FGAT embeddings |
| 2 | **Suggest Outfit Compatibility** | `/compatibility` | Search and select fashion items, analyze their compatibility score (0–100 %), and receive outfit-completion suggestions |
| 3 | **Find Most Similar Outfits** | `/similar` | Browse all outfits in the dataset, select one, and retrieve the top-10 most similar outfits by cosine similarity |

## Quick Start

### Prerequisites

- Node.js ≥ 18
- The backend server running (see [hcmus-master-is-ai-fgat-be](https://github.com/1653072/hcmus-master-is-ai-fgat-be))

### Install & Run

```bash
# 1. Install dependencies
npm install

# 2. Configure backend URL (copy example and edit if needed)
cp .env.local.example .env.local
# Default is http://localhost:5000 — no changes needed for local dev

# 3. Start development server
npm run dev
```

The app is available at **http://localhost:3000**.

### Other Scripts

```bash
npm run build   # Production build
npm run start   # Start production server
npm run lint    # ESLint check
```

## Backend URL Configuration

The frontend proxies all `/api/*` calls to the backend via `next.config.ts`. Switching environments requires only one env variable:

| Environment | `.env.local` setting |
|-------------|----------------------|
| Local dev | `BE_URL=http://localhost:5000` *(default)* |
| Render (production) | `BE_URL=https://your-app-name.onrender.com` |

**For Vercel deployment:** add `BE_URL` as an environment variable in *Settings → Environment Variables* in your Vercel project dashboard. No code changes needed.

How the proxy works:

```
Browser → /api/recommend  →  Next.js rewrite  →  BE_URL/recommend
```

## Design System

|-------|-------|-------|
| `--bg` | `#f0ebe3` | Page background (warm parchment) |
| `--surface` | `#e8e1d6` | Cards, panels |

| `--accent` | `#c8a96e` | Gold — headings, active states, CTAs |
| `--accent2` | `#8b5e3c` | Dark brown — hover accents |
| `--text` | `#1a1510` | Body text |
| `--muted` | `#7a6f65` | Secondary / placeholder text |
| `--green` | `#5a8a6e` | Score indicators |

- **Fonts:** `Cormorant Garamond` (serif headings) · `DM Mono` (body/mono)
- **Category colors:** integer IDs → deterministic palette via `getCategoryColor(id)` in `constants.ts`
- **Tailwind CSS v4** via `@tailwindcss/postcss`

## Notes

- Item titles are in **Chinese** (from the AliExpress fashion dataset used to train H-FGAT).
- Item images are served from `gw.alicdn.com` (AliCDN); a fallback 👗 icon is shown if an image fails to load.
- The backend on Render free tier has a **cold-start delay of ~30–60 s**. The API timeout is set to 30 s; if requests fail on first load, try again after the backend warms up.
- User IDs, outfit IDs, and item IDs are all **strings** in the backend responses.
- Category IDs are **integers** (e.g., `3`, `10`, `16`, `43`). The frontend maps them to colours using a rotating palette.
